INTERACTION FLOW
═══════════════════════════════════════════════════════════════════════════════

1. OPENING THE MENU
   User Action: Right-click (button 3) anywhere in text area
   Result: Menu appears at cursor coordinates (x, y)

2. HOVERING OVER ITEMS
   User Action: Move mouse over menu item
   Result: Item background changes to amber, text darkens

3. CLICKING AN ITEM
   User Action: Left-click on enabled menu item
   Result: Action executes (copy/select all/clear), menu dismisses

4. DISMISSING THE MENU
   Options:
   - Click outside the menu → Menu disappears
   - Press Escape key → Menu disappears
   - Click a menu item → Action executes, menu disappears
   - Right-click again → New menu replaces old menu


KEYBOARD SHORTCUTS (Also shown in menu)
═══════════════════════════════════════════════════════════════════════════════

Ctrl+C / Cmd+C .......... Copy selected text to clipboard
Ctrl+A / Cmd+A .......... Select all text in output
Esc ..................... Clear selection or dismiss menu

      └────────────────┘         └────────────────┘
      ↑                                           ↑
      Shifted left                     Shifted up


MEASUREMENTS
═══════════════════════════════════════════════════════════════════════════════

Menu Width: ~32 characters (including border)
Menu Height: Number of items + 2 (for borders)
Item Height: 1 line per item
Item Padding: 0 vertical, 2 horizontal (spaces on each side)
Border: 1 character width (Textual solid border)


ASCII BOX CHARACTERS USED
═══════════════════════════════════════════════════════════════════════════════

The menu uses Textual's built-in border rendering, which typically uses:

│ │  Vertical edges
└─┘  Bottom edge

NOTES
═══════════════════════════════════════════════════════════════════════════════

1. Menu appears exactly where the user right-clicks
2. Menu is always on the overlay layer (appears above all content)
3. Menu automatically positions itself to stay within screen bounds
4. Only one context menu can be active at a time
5. Menu dismisses automatically after executing an action
6. All actions are also available via keyboard shortcuts
7. Disabled items are clearly indicated with dimmed text
8. Hover feedback is immediate and visually distinct
